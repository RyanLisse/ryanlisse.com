## Overview

Engram turns agent memory from a prompt hack into infrastructure. It stores decisions, habits, and learned context across sessions so coding agents can resume work with continuity instead of starting from zero each time.

## What It Solves

Most agent workflows are stateless in practice. The result is repeated onboarding, forgotten context, and brittle output quality from one session to the next.

Engram introduces a memory layer that helps agents:

- Persist relevant context across runs
- Recover prior decisions and constraints quickly
- Share learnings across parallel agents
- Reduce redundant prompting in long-lived workflows

## Core Capabilities

- **Cross-session memory** for ongoing engineering work
- **Structured recall** so agents retrieve the right context at the right time
- **Procedural memory tracking** for workflows, not just facts
- **Multi-agent compatibility** with shared memory patterns

## Outcome

Engram is part of a broader push toward agent-native systems where continuity matters. It makes multi-agent development more reliable by giving the system a memory substrate instead of relying on the operator to restate context manually.
